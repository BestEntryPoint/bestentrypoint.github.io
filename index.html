    What is Programming indeed?    

Programming is when we make a machine do what we want by just writing some text.


Such texts are called programs (or codes) and like any other texts are written in languages. The languages the programs are writen in are named programming languages.

We here will learn how to write the programs in such amazing programming languages as Kotlin and Swift.


Why Kotlin and Swift?
 This is because they both are new, super effective programming languages making it easy to write programs that are incredibly fast.

Plus, fortunately, these languages are almost the same.

With Kotlin we can write programs for all Android devices.


With Swift we can write programs for all Apple devices.


So it would be unbelievably awesome to know both languages.

I predict that from now on we will see more and more people eager to learn both the languages, and of course also, to learn them simultaneously.

Commanders
 As we already know, programming is a wonderland where we can make machines do the works we want by just writing some texts named codes.


But how, exactly, should codes be arranged inside?

Right now, we will learn that.

In programming, we command the machine only by Commanders:


So it is the Commanders that make the machine do the works that we want. They are like our helpers.

Interestingly, these Commanders (our helpers) exist only in the mind of the machine:


We cannot feel them by our hands. But they really exist and can make the machine to do the works we want.

So, thankfully, we can explicitly interact with Commanders.

Each such a Commander has a name:


and there is a lot of Commanders with different names.

Importantly, one Commander can only make the machine do one specific work. One Commander, one work.

Just like on the street, when we need specific persons we might call (invoke) them by their names:


in programming, when we need specific Commanders to make the machine do specific works we also can call (invoke) them by their names.

Of course, in programming, we can call (invoke) Commanders only by writing their names in code:


Let's look again at our familiar code:
  fun main()  {
    print(5)
}		                       

print(5)
 
 		  
The print word in the code is namely the name of a Commander:

  fun main()  {
    print(5)
}		                        print(5)
 
 		  

And we in the code, by writing the Commander’s name print, call (invoke) this Commander to make the machine do some specific work.

So, specifically, the Commander named print makes the machine do the following work:

   — print makes the machine take our number 5 being inside the brakets ( ) and print (display) this number as a result (5) on the screen:
  fun main()  {
    print(5)
}		                       

print(5)
 
 		  
         
  5   5  

So the full name of this Commander is print().
